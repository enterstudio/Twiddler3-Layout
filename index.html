Looking for a chorded-keyboard layout standard

Why should we learn some chording layout that makes sense only to the author? A better idea would be to take a part in the build process. And if it's too late, having a chance to read the story is the next best way to get the idea.

Every single decision should be clearly justified to avoid having to keep tabs on unclear points requiring further explanation. And this is not the case even in the TabSpace Guide by Brandon Craig Rhodes, the original author of what stands for a standard layout of this device.

A systematic study of all the possible combinations should help at deciding new assignment rules. Some grouping methods should be defined to replace "simply good-looking patterns". To be more easily accepted as an effective improvement, these rules should make sense when looking at the big picture.

Supported chords

The Twiddler ORML notation does not support chords that would require pressing more than one button on the same row.

The whole set of possible chords can be seen as a group of 4 sub-sets. Each sub-set of 3x3 chords from the 3 columns of the 3 lower rows would have to be combined with one of the 4 ORML layers of the first row to cover all the combinations.

Not covered here, less practical (yet usable) combinations of 2 modifiers on the first and second row of the 3x3 sub-pads allows 24 more mappings.

Twiddler 3 Layout

As last serious mapping approach was made in 1999, it's about time to go forward and try to build something new. A better graphical representation should help at making a decision for some mapping choices.

Best mnemonic metaphores come more easily when categories of symbols are grouped in a way their disposition builds some kind of recognizable contour. We can see this at work in first and second column of the last Twiddler 3 Layout row:
- Col1L9x with {?} (!) [_]
- Col2L6x with <^> /|\

Layout Rows

The layout pads are organized in 4 rows of 11 columns. Each row corresponds to the 4 possible ways the index finger is used with buttons of the first row BS, Space and Tab to serve as modifiers like this is the case on a standard keyboard with Shift, Control and Alt.

All the 3 by 3 pads can be refered to by the Twiddler ORML notation: 

O The No modifier layer
Key strokes of this layer are generetad without the index finger.

R The Right modifier layer
The BS Right first row button is pressed by the index finger.

M The Middle modifier layer
The Space Middle first row button is pressed by the index finger.

L The Left modifier layer
The Tab Left first row button is pressed by the index finger.
Layout Columns

What the index finger does on the first row of the 4x3 full pad, the other fingers can do on their respective row of the 3x3 sub-pad to introduce more layers. The remaining buttons can then be mapped individually or by pairs.

Each 3x2 sub-pad is composed of a limited number of mappable single buttons or pairs.

Why Num-Caps-Scroll Locks should be in lower Rxx
So that the easy no-mod O-2 (row O, col 2) pad can be used for the following frequently used punctuation symbols , : ; and ' " ` accents and quote.

  • Why 2L 3L have been moved to Num-combo
  • TabSpace letters arrangement takes care of english letter usage frequency.
  • Most 2 and 3 combined letters are placed over their constitutive letters location but they belong to different modifier-combos:
  • If combinations such "an", "in", "qu" or "the" are obtained by pressing the letters they contain , it is not the case for "ch", "nd", "of" or "ing" which refer to letters from 2 different modifier combinations.
    • The good part in this loose rule is that each button of these chords involve the finger used to produce those individual letters.
    • The not so good is that it relies on some unspecified rule that would have to "explain" how the selected modifier combo works to produce the result.
  • For instance, how "ing" is obtained by combining "i", "n" and "g"? - "i" and "n" are on the first column of "0 single" layer - "g" comes from "M single" layer.
Note
The 4x11 layout can be further extended to include the less practical, but still usable, combinations of 2 modifiers on the first and second row of the 3x3 sub-sub-pads to obtain 24 more mappings.
